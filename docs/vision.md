# FamilyMemoryOS 项目愿景与边界

## 1. 项目愿景

FamilyMemoryOS 希望成为一个本地优先、隐私优先、长期可维护的家族数字记忆框架。

它服务的不是“高拟真扮演”，而是“记忆保存、关系组织、资料检索与受约束的纪念性交互”。

项目希望帮助家庭把分散在聊天、照片、书信、录音转写、日记与回忆中的资料，逐步整理为一个可管理、可检索、可持续维护的数字记忆空间。

---

## 2. 当前阶段的核心目标

当前阶段只关注以下问题：

- 如何表示家族人物与关系
- 如何保存人物事件与家族时间线
- 如何导入本地文本资料
- 如何为后续检索与记忆片段做准备
- 如何建立清晰、稳健、低复杂度的工程骨架

---

## 3. 当前阶段不追求的目标

当前阶段不追求：

- 高度拟真的人格模仿
- 语音复刻
- 3D 数字人
- 自动多模态全解析
- 云同步平台
- 公共分享平台
- 情感替代型产品

---

## 4. 关键原则

### 本地优先
项目默认支持本地部署与本地资料保存。

### 隐私优先
家族资料默认视为敏感数据，系统设计不应默认依赖公网。

### 可追溯
生成结果应尽量能说明来源或依据等级。

### 可修正
所有提取、整理、建模结果都应允许人工修正。

### 渐进式建设
先完成可靠的数据层和工程骨架，再逐步实现更复杂的能力。

---

## 5. 工程策略

项目工程策略为：

- 先小后大
- 先稳后全
- 先骨架后能力
- 先本地后扩展
- 先数据结构后人格生成

---

## 6. 当前最小闭环

FamilyMemoryOS 当前最重要的最小闭环是：

人物 → 关系 → 事件 → 文本资料 → 记忆片段 → 检索问答

在这个闭环跑通之前，不应优先投入复杂人格交互能力。
